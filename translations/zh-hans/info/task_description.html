你的任务是将不同类型的血液分配给需要输血的病人，同时考虑到每种血型的相容性限制。 血液供应并不总能满足所有需求。您将获得每种血型的数量和每种病人的需求。 您的目标是在遵守配血规则的前提下优化分配可用血液。 请记住，可能会有多个最佳解决方案，即最大限度地利用现有血液供应的方案。

兼容性限制：

输入：每种血型的血液数量（blood_avail）和血液需求（blood_needs），作为{% if is_js %}地图对象{% else %}字典(dict){% endif %} ，字符串键 (str{% if is_js %}ing{% endif %}) 和整数值 ({% if is_js %}number{% else %}int{% endif %}).

输出：输出结果将是每种血型的血量分布，如{% if is_js %}地图对象的地图对象{% else %}字典的字典(dict){% endif %} ，字符串键 ( ) 和整数值 ( ) 。str{% if is_js %}ing{% endif %}) 和整数值 ({% if is_js %}number{% else %}int{% endif %}).

如何使用： 该任务的重点是根据兼容性有效分配可用血量，以满足病人的需求。 函数 "distribute_blood "会计算适当的血液分配，同时尊重兼容性限制。 通过完成本任务，您将深入了解高效血液分配算法及其实际应用。

前提条件：
0 ≤ 血液数量；
0 ≤ 血液需求。
