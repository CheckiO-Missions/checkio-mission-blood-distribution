Państwa zadaniem jest dystrybucja dostępnej krwi różnych grup do pacjentów wymagających transfuzji, biorąc pod uwagę ograniczenia dotyczące zgodności każdej grupy krwi. Zapasy krwi nie zawsze są wystarczające, aby zaspokoić wszystkie potrzeby. Otrzymają Państwo informacje na temat ilości każdej grupy krwi i potrzeb każdego pacjenta. Państwa celem jest optymalne przydzielenie dostępnej krwi, z poszanowaniem zasad zgodności. Proszę pamiętać, że może istnieć wiele optymalnych rozwiązań, które wykorzystują maksymalną możliwą ilość dostępnej krwi.

Ograniczenia dotyczące zgodności:

Dane wejściowe: Ilość krwi(blood_avail) i zapotrzebowanie na krew(blood_needs) dla każdej grupy krwi jako obiekty mapy {% if is_js %}{% else %} słowniki (dict){% endif %} z kluczami łańcuchowymi (str{% if is_js %}ing{% endif %}) i wartościami całkowitymi ({% if is_js %}number{% else %}int{% endif %}).

Dane wyjściowe: Wynikiem będzie rozkład ilości krwi dla każdej grupy krwi jako {% if is_js %}obiekt mapy obiektów mapy{% else %}słownik słowników (dict){% endif %} z kluczami łańcuchowymi (str{% if is_js %}ing{% endif %}) i wartościami całkowitymi ({% if is_js %}number{% else %}int{% endif %}).

Jak jest używany: Ta misja koncentruje się na efektywnej dystrybucji dostępnych ilości krwi w celu zaspokojenia potrzeb pacjentów w oparciu o zgodność. Funkcja `distribute_blood` oblicza odpowiednią alokację krwi, przestrzegając ograniczeń zgodności. Wykonując tę misję, uzyskają Państwo wgląd w efektywne algorytmy dystrybucji krwi i ich praktyczne zastosowania.

Warunki wstępne
0 ≤ ilość krwi;
0 ≤ zapotrzebowanie na krew.
